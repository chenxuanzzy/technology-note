# 1 - JavaScript 基本介紹

  • 有國際標準化組織 ECMA 的統一規範
  • 與 Ajax 等技術打造與以往不同的單一網頁 SPA（Single Page Application）

    過去網頁上通常都是第一次存取時將檔案完整讀取，但是有了 SPA 之後當只有某區塊需要被更新可以不用每次重新讀取整個網頁，只重新讀取單一區塊即可。

  • JavaScript 的四個特徵
    • 腳本語言
    • 直譯語言，依照執行順序來轉譯為機器語言
    • 可應用於各版本瀏覽器
    • 多種功能的構成
  • DevTool 的應用與介紹

# 2 - 基本語法

<script type="text/javascript">

//腳本撰寫區塊

</script>

  • 通常都是使用 HTML5 內的撰寫方式，文字編碼預設為使用 UTF-8

  • 撰寫<script>的位置

    • <body> 內，封閉</body>之前

  • <script>可以出現多次，因為最後會彙整讀取

  • 寫成外部檔案的優點

    • 程式版型分離化，函式可重複利用
    • 程式外部化可有效簡化 html 內容，好閱讀
<script type="text/javascript" src="scripts/hello.js"></script>
// hello.js 為外部 js 檔案

  • 不符合規定的外部內嵌寫法

  • 當有設定 src 的外部檔案時的的標籤內的內容會被忽略

    <script type="text/javascript" src="scripts/hello.js">
    windows.alert('Hello World!'); //此行會被忽略
</script>

  • 無法執行 JavaSciprt 時可用<noscript>標籤取代

    瀏覽器可設定封鎖 JavaScirpt 的使用

  • 錨點(Anchor)嵌入

<a herf="JavaScript:window.alert('Hello, World');>"

  • 註解 Comment Out、取消註解 Comment In

    • When I say "comment out", does it mean to uncomment something or comment it? (opens new window)

  • 變數宣告

    • var 全域變數、let const 區域變數

      不允許變數重複：var 如果重複宣告最後一個宣告的一樣名稱的變數則會被覆蓋，而 let 則會報錯（Identifier 'msg' has already been declared）

    var msg = 'Halu'
var msg = 'My Name'
console.log(msg) // My Name

  • 統一命名方式

    • 變數/函數名稱 => camelCase 方法：

      開頭單字的第一個字元小寫，之後的單字第一個都大寫　 ex. lastName、myName

    • 常數名稱 => Pascal 法：

      所有單字的第一個字元都大寫 ex. LastName、MyName，常數為了好分別在宣告時都使用全大寫

    • 類別（建構子）底線法：

      單字間以底線做連結　 ex. last_name、my_nam

  • 資料型別

    • JavaSciprt 的資料型別分為基本型和參考型，兩者差異於變數值存入的方式

      • 基本型：變數可以直接填入變數值
      • 參考型：變數以儲存參考值，以儲存「值」的記憶體位置

    • 常值（Literal）是指資料型別中時劑處存值的表示方法

      • 數值（Number）=> 整數型別、浮點數型別

        • 10 進制 0~9
        • 2 進制 0b, 0~1
        • 8 進制 0o, 0~7
        • 16 進制 0x, 0~9, A~F

      • 字串（String）=> 單引與雙引

      • 樣板字串（Template Strings）

        • 在字串中嵌入變數
        • 支援多行字串
        • 使用反引號（Backquote）
          let age = 22
let str = `I'm ${age} old.`
console.log(str) // I'm 22 old.
