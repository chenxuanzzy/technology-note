Alpha Camp 筆記 - 學期1-S1 寫程式的起點

Chen Xuan
2021-06-10
  • Alpha Camp 筆記

這次報名參加了 Alpha Camp 學期課程，而學期一的課程比較簡單，內容上或許比較少，但記錄的都是一些平時不會特別想到的東西，怕忘記所以寫成筆記來提醒自己複習用。

# 電腦與程式

電腦是透過程式碼的指令，經由以下的流程來完成需求。

輸入(input) > 儲存(storage) / 處理(processing) > 輸出(output)

我覺得其實這就像是一個人類大腦，需要輸入知識，儲存後做處理讓大腦吸收，再輸出應用知識，想成這樣或許更容易理解。

# 高階語言和低階語言

這部份以前計算機概論有學過，高階語言就是人類看得懂的語言，而低階語言是電腦看得懂的語言，低階語言又包含機器語言 (opens new window)跟組合語言 (opens new window)，這個觀念知道就好，因為寫程式上不會特別做討論，但別人問的時候至少要回答得出來高階低階的差異。

# 初步認識 JavaScript

你覺得熱狗跟狗一樣嗎？ 那 Java 跟 JavaScript 一樣嗎？

每次跟別人說自己在寫 JavaScript 的時候，有些人都會以為就是 Java，但其實兩者都根本就是不同東西，JavaScript 是前端三大重點之一，現在我們看到的網頁就是由 Html、CSS、JavaScript 所組成的，而當初會這樣取名字是因為那個年代 Java 正火熱，為了搭上潮流的創作者 Brendan Eich 不到十天寫出了 JavaScript 這個寫瀏覽器的語言，然後取名為跟 Java 很像的 JavaScript。但也因為開發時間短，有很多奇怪的地方，像是型別轉換等等，為了讓大家有規則的去使用這個語言所以有一個叫做 ECMA 國際標準化組織把規範列出來，從以往到現在最常使用的 ES6，最新版本的 ES7、ES8 都是 ECMA 訂定出來的規範。

# JavaScript 變數介紹

上述提到它會隨便轉換型態，屬於弱型別 (opens new window)語言，它不像 Java、C++ ..的強型別語言，所以我們在一開始宣告變數（Variable）的時候要明確地給變數作用域（Scope），乾淨的變數也不會互相干擾。

# let

let luckyNum = 7;

以往我們可能會使用 var 去做宣告，但 var 的範圍是全域，全域看似沒什麼問題但當兩個變數名稱相同時很容混淆外， JavaScript 又是弱型別語言，造成值的改變，當 ES6 出來後工程師們以這個為規範去做使用，let 比較像是小區域的宣告（大範圍宣告也可以），就像是每個小泡泡裡面都有自己的專屬顏色，因為外圍的泡泡包著所以不會干擾到其他泡泡內的顏色，也就不會混在一起。

# const

const arr = ["10", "9", "8", "7", "6"];

當我們這個值不會隨意改變的時候通常就會使用 const ，最常宣告常數應用，像是陣列（Array）、物件（Object），假設我們要記錄一群朋友的年紀，這種時候就可以用 const 宣告，因為年紀不會隨便增加改變。

# 命名方式

命名的方式對於未來回頭 code review ，或是給下一個人看，都是很重要的一件事，也有幾個要注意的命名事項，以下幾點是我覺得重要的

  • 特殊符號

可使用 $、__ 當做命名字首，但如果沒有特殊目的的話還是建議用英文當變數字首。

let $luckNum = 7;
let _myPhone = '0983';
  • 駝峰式（Camel Case）的命名方式

在 JavaScript 中有區分大小寫，儘管拼起來一樣，如果大小寫不一樣那就會是不同的變數，而駝峰式命名就是第一個單字小寫，後續單字字首大寫，閱讀上會比較方便。

let myLuckyNum = 7;
let MyluckyNum = 777;
  • 不可使用的命名方式
  1. 數字開頭 let 555 = ...
  2. 烤肉串（Kebab-case）命名 let my-phone = ..

# 運算子（Operator） (opens new window)

運算子有以下四種算術運算子、賦值運算子、比較運算子、邏輯運算子

# 一、算術運算子（arithmetic operators）

言簡意賅的說就計算的加減乘除，但這邊有個額外要注意的是放在變數前和後的差異

let num = 7
console.log(num++); //印出 7 後把值 +1

let num = 7;
console.log(++num); //印出前將 7+1 印出 8

# 二、賦值運算子（assignment operators）

就是在=前加上加減乘除，這個我們很常拿來使用

let num = 5;
let age = 10;
num += age; //同等於 num = num + age 故 num = 15（其他運算使用方式一樣）

# 三、比較運算子（comparison operators）

比較兩者的值回傳的布林值是 true 還是 false，比較兩者值會因為等於符號回傳不一，以下範例為寬鬆跟嚴格差異，建議在撰寫上都使用嚴格用法

1 == '1' //true，不寬鬆用法
1 === '1' //false ，完全等於，嚴格用法

1 != '1' //false，不寬鬆用法
1 !== '1' //false，完全不等於，嚴格用法

# 四、邏輯運算子（logical operator）

有 &&（and）、||（or）、!（非）

  • && 如果前後都是 true，回傳 true，有一方是 false 則回傳 false
true && true //回傳 true
true && false //回傳 false
false && false //回傳 false
  • || 只有一方為 true 就回傳 true
true || true //回傳 true
true || false //回傳 true
false || false //回傳 false
  • !（非） 將布林值轉換相反
!true //回傳 false
!false //回傳 true

# 演算法的定義

在數學與電腦科學領域中，演算法是一組有限的指令，這組指令已經被充分定義，可以在電腦上實作，通常用於解決問題或執行運算，從定義來說演算法就是指令的集合體。

演算法和數學裡的函式（Function）說起來是相通的，而回到一開始的電腦與程式說到輸入、儲存/處理、輸出，演算法或函式的重點在於如果環境和輸入的內容不變，每次執行同一組指令的時候，就會得到相同結果。

Input   >         Algorithm        >  Output
外部輸入 > 給電腦的指令的程式執行步驟 > 執行結果

# 額外補充

# 提升 Hoisting (opens new window)

MDN 這邊有解釋道

提升看起來是單純地將變數和函式宣告，移動到程式的區塊頂端，然而並非如此。變數和函數的宣告會在編譯階段就被放入記憶體，但實際位置和程式碼中完全一樣。而這樣做的優點是：可以在程式碼宣告該函式之前使用它。

假設我們使用 var 宣告變數，變數在以下會被提升，賦值則停留在原地。

console.log(x); //undefiend
var x = 10;

也同等於

var x;
console.log(x); //undefiend
x = 10;

# 樣板字串（template strings） (opens new window)

利用反引號的方式引入字串和數值，也可以做字串內的運算，語法為 ${} 像是

let age = 23;
consoele.log(`你今年滿 ${ age } 歲囉`) //你今年滿 23 歲囉
