# 靜態頁面 HTML 與 CSS

像我的部落格就是使用靜態網頁去建置，靜態與動態最大的差異是有沒有後台，普遍看到 WordPress 就是動態，可以讓作者觀看後台瀏覽文章數據或是觀站次數等等功能，而我建置的部落格就沒辦法。

# HTML 超本文標記語言

HTML ( Hypertext Markup Language) 叫做超文本標記語言，是 Markup Language 不是 Programing Language

# CSS 階層樣式表

CSS ( Cascading Style Sheets ) 稱為階層樣式表，網頁所呈現的顏色、樣式都是由它來設定

  • border 框線
  • padding 框線到內容間的空白

以下用法為常見的寫法，也可另外寫 padding-top 或是 padding-bottom等用法。

padding: 10px 10px 10px 10px;
          上   右    下  左
padding: 15px 15px;
          上   下
  • margin 框先到其他元素的距離 ( margin 用法同上)

打開 DevTool 裡的 Computed 可以實際操作

# 使用者故事

使用者故事簡單說就是站在使用者的角度，會希望網頁幫你完成什麼事，而把核心條件列出來有利於之後的開發，太抽象的說法其他人也不懂。以課程作業來說，如果今天要撰寫一個電子郵件功能，你會希望有哪些功能？以下是我的使用者角度。

撰寫功能

  • 作為一個使用者，我可以在尚未寄出之前編輯我的信件內容且自動儲存成草稿
  • 作為一個使用者，我可清楚知道哪些欄位是收件人、主旨..
  • 作為一個使用者，我可以將信件多人轉寄 (不用重複操作)
  • 作為一個使用者，我可以附帶檔案或是調整寄件內容版面等等操作
  • 作為一個使用者，我可以輕鬆存取已寄送過的收件者名單（快取名單）

收信功能

  • 作為一個使用者，我可以在列表中看到所有郵件。
  • 作為一個使用者，我可以將列表中的所有信件做排序或分類、標記
  • 作為一個使用者，我可以刪除、封存、復原信件
  • 作為一個使用者，我可以清楚的知道是誰寄送過來的以及寄送時間
  • 作為一個使用者，我可以自訂停止接收哪些不想要的接收到的郵件

額外功能

  • 作為一個使用者，我可以快速的將所有郵件標記、已讀、分類
  • 作為一個使用者，我可以將其他收件地址轉為同一個信箱（例如學校官方 Email 可以改成 Gmail 當接收口）
  • 作為一個使用者，我可以設定自己想要顯示的資料列表筆數
  • 作為一個使用者，我可以隨意更改我喜歡的背景樣式
  • 作為一個使用者，我可以看到郵件聯絡人是否在線上

# 本週心得

S2 就像是牛刀小試，寫了一個猜數字的程式，在建立流程圖上我蠻沒有耐心的（看其他同學也有跟我一樣的感受），很多時候都是直接開始寫程式，再回頭寫流程圖，但有流程圖再寫程式邏輯上的建構會比較清楚。我在工作上如果看到前輩寫的流程圖再去看他的 Code 邏輯確實清晰很多，這部分我還需要再多加磨練。而 S3 介紹網頁的部分，補充了一些與設計上的溝通，這部份也特別記下來，設計與前端可以說是息息相關呢！
