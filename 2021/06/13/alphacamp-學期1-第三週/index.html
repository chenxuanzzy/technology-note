# 語義化元素（Semantic element）

顧名思義就是直接表達，像是一個網頁的架構除了可以用<div>去表達以外，也可以使用<header>、<nav>、<main>、<section>，雖然肉眼上沒有太大的差異，但是對於瀏覽器的解析來說語義化是最適合不過的，另外語義化的寫法對於 SEO (opens new window) 比較適合。

# 超連結<a></a>

當我們使用超連結標籤時會特別加上target=_blank，代表原本視窗不會被關掉而另外開啟一個新的視窗，完整的寫法為

<a herf="https://github.com/chenxuanzzy" target="_blank"></a>

# Flex 佈局

可以把一個它想像成是一個容器，將它變成容器的話我們通常是用display:flex，外容器的排列有很多種，這篇 (opens new window)文章有詳細介紹，而容器內要排列的話最常使用的為以下三種（之後會做一個總匯整理，這裡單純紀錄一下）

  • display: block 區塊元素

占用容器的寬度，使相鄰的擠到下一行區，如同中文意思，它就是一區塊一區塊的呈現

  • display: inline 行內元素

占用容器的空間框度皆由內容決定，相鄰的元素會因為空間擠到下一行，像是魚貫的呈現

  • dislplay: inline-block 混合用法

用 inline 的排版方式但具有 block 屬性

# 元素實際大小 box-sizing

我們看 box model 的時候是由 padding、margin、border 所組成，但計算上卻有些微的差異，瀏覽器在預設的情況下會把三種設定所加入，這會造成呈現上的差異，所以一開始在切版會建議在全局主動加上 box-sizing: border-box

# 本週心得

S4 介紹了網頁上的語法以及瞭解 flex ，在切版時我都直接使用<div>，雖然目前不用考慮到 SEO 的部份（不管是工作上或是自我練習），但瞭解語義化對於瀏覽器的解析是比較好的做法，往後的自己會注意 Html 標籤上的應用。課程中也寫了幾個簡單的作業，CSS 的部分我覺得多寫幾次就會上手。S5 的部份有額外寫了三份的期末選修，其中讓我卡比較久的是 Q1 的當螢幕超過 820px 時會往內縮，一開始用 @media 去下，後來才發現不需用特別使用 @media 設定斷點，直接在元素上下 max-width 就可以了（就是自己想太多XD）
